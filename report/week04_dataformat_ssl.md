# JSON, XML, YAML 비교 정리

## 데이터 포맷이란?

- 시스템에서 데이터를 저장하거나 입출력할 때 사용하는 데이터 구조, 형식, 규칙을 의미함

---

## 1. JSON (JavaScript Object Notation)

- **형식**: 키-값 쌍으로 이루어진 사람이 읽기 쉬운 텍스트 기반 데이터 포맷
- **예시**:
  ```json
  {
    "student": [
      {
        "id": 1,
        "name": "김철수",
        "age": 18,
        "hobby": "농구"
      },
      {
        "id": 2,
        "name": "박영희",
        "age": 17,
        "hobby": "십자수"
      }
    ]
  }
  ```
- **특징**
  - 다양한 자료형 제한 없이 변수 값 표현에 적합
  - 순서 인덱스를 기준으로 데이터 접근
- **장점**
  - 문법이 단순하고 명확하여 거의 모든 언어에서 쉽게 사용 가능
- **단점**
  - 콤마, 따옴표 누락 등 문법 오류에 취약
  - 주석을 지원하지 않음

---

## 2. XML (eXtensible Markup Language)

- **형식**: 태그 기반의 마크업 언어로, 데이터 구조와 의미를 기술
- **예시**:
  ```xml
  <student>
    <id value="1">
      <name>김철수</name>
      <age>18</age>
      <hobby>농구</hobby>
    </id>
    <id value="2">
      <name>박영희</name>
      <age>17</age>
      <hobby>십자수</hobby>
    </id>
  </student>
  ```
- **특징**
  - 트리 구조로 계층적 데이터 표현 가능
  - 데이터와 메타데이터를 함께 표현 가능해 의미 부여에 강점
  - 소프트웨어/하드웨어 독립적으로 데이터 처리 가능
- **장점**
  - 복잡한 메타데이터 선언 가능 → 정보 표현에 효과적
- **단점**
  - 태그가 많아 데이터 크기가 커질 수 있음
  - JSON 등장 이후 활용 빈도 감소
- **사용 예**
  - 게임 설정, 복잡한 데이터 구조에 적합

---

## 3. YAML (YAML Ain't Markup Language)

- **형식**: 들여쓰기와 띄어쓰기로 계층과 데이터를 표현하는 사람이 읽기 쉬운 데이터 직렬화 포맷
- **예시**:
  ```yaml
  Servers:
    - name: Server1
      administrator: Kim
      created: 20050103132749
      status: active
    - name: Server2
      administrator: Lee
      created: 20210101000000
      status: active
  ```
- **특징**
  - 들여쓰기가 매우 중요하며, Map(key-value) 구조 사용
  - 복잡한 객체나 참조 정의에 적합
- **장점**
  - 가독성이 매우 좋음
  - 복잡한 데이터 구조를 쉽게 표현 가능
- **단점**
  - JSON보다 문법이 복잡해 파싱과 생성 속도가 느림

## HTTPS & SSL 인증서 기초 정리
### HTTPS란?
- HTTP 프로토콜의 보안 강화 버전으로, TLS/SSL 암호화를 통해 데이터 전송 시 보안을 제공함

- 웹 브라우저와 웹 서버 간 통신을 암호화하여 도청 및 변조를 방지

- TLS/SSL 인증서를 기반으로 통신 상대방의 신원을 검증함

- 기본 사용 포트: 443번 (HTTP는 80번)

### HTTPS가 중요한 이유
- 데이터 보호: 네트워크 상에서 정보가 평문(일반 텍스트)으로 전송되지 않음

- 스니핑 방지: 패킷을 가로채도 암호화되어 의미 없는 문자로만 보임

- 중간자 공격 차단: ISP나 중간자가 임의로 콘텐츠를 삽입하는 공격 방지

- 사용자 신뢰 증대: 보안 연결이므로 사용자 신뢰 확보 가능

### SSL 인증서란?
- 웹 서버에 설치되는 데이터 파일

- 웹사이트 공개 키와 신원 정보(도메인, 소유자 등)를 포함

- 클라이언트가 서버와 안전하게 통신할 수 있도록 도와줌

### SSL이란?
- Secure Sockets Layer의 약자로, 인터넷 통신 암호화 및 서버 신원 확인 프로토콜

- 현재는 TLS(Transport Layer Security)로 발전되었으나, SSL이라는 명칭이 여전히 널리 사용됨

- HTTPS 주소가 붙은 모든 웹사이트에서 사용

### SSL 인증서에 포함된 정보
- 인증서가 발급된 도메인 이름

- 소유자 정보 (개인, 조직, 장치)

- 인증 기관(CA, Certificate Authority) 정보

- CA의 디지털 서명

- 관련 하위 도메인(있는 경우)

- 인증서 발급 및 만료 날짜

- 공개 키 (비밀인 개인 키는 서버가 별도로 보관)

### SSL이 필요한 이유
- 데이터 암호화: 공개키-개인키 쌍으로 안전한 암호화 통신 수행

- 서버 인증: 클라이언트가 올바른 서버(도메인 소유자)와 통신 중임을 확인 → 도메인 스푸핑 방지

- HTTPS 구현: HTTPS는 SSL/TLS 인증서 없이는 작동하지 않음

- 사용자 신뢰 및 보안 강화

### SSL 인증서 발급 방법
- 도메인 소유자가 신뢰받는 **인증기관(CA)**에서 인증서 발급 신청

- CA가 소유권 확인 후 인증서 발급 및 디지털 서명

- 발급 받은 인증서를 서버에 설치하여 HTTPS 통신에 사용
